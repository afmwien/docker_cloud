SERVICE (services/image_finder.py)
===================================
import requests
from django.conf import settings

def search_image(website_url, reference_image_path, threshold=20):
    with open(reference_image_path, "rb") as f:
        response = requests.post(
            f"{settings.IMAGE_FINDER_API_URL}/search",
            headers={"X-API-Key": settings.IMAGE_FINDER_API_KEY},
            files={"reference": f},
            data={"url": website_url, "threshold": threshold},
            timeout=120,
        )
    return response.json()

def take_screenshot(website_url, mode="desktop", scroll_to=0, full_page=False):
    response = requests.post(
        f"{settings.IMAGE_FINDER_API_URL}/screenshot",
        headers={"X-API-Key": settings.IMAGE_FINDER_API_KEY},
        data={"url": website_url, "mode": mode, "scroll_to": scroll_to, "full_page": full_page},
        timeout=120,
    )
    return response.json()

def save_base64_screenshot(b64_data, output_path):
    """Base64-Screenshot-Daten als PNG-Datei speichern."""
    import base64
    with open(output_path, "wb") as f:
        f.write(base64.b64decode(b64_data))

def download_screenshot(screenshot_path):
    """Fallback: Screenshot per GET herunterladen (nur solange Container lebt)."""
    response = requests.get(
        f"{settings.IMAGE_FINDER_API_URL}{screenshot_path}",
        headers={"X-API-Key": settings.IMAGE_FINDER_API_KEY},
        timeout=30,
    )
    return response.content

VERWENDUNG IM VIEW
===================
from services.image_finder import search_image, take_screenshot, save_base64_screenshot

# --- Bildsuche ---
result = search_image(
    website_url="https://www.maisai.at/fotos/",
    reference_image_path="/pfad/zum/bild.jpg",
)

if result["success"]:
    print(result["match"]["url"])       # URL des gefundenen Bildes
    print(result["match"]["distance"])  # 0 = identisch
    print(result["match"]["position"])  # {"x": 1254, "y": 2354}
    print(result["match"]["size"])      # "400x400"
    print(result["images_checked"])     # Anzahl geprüfter Bilder

    # Screenshot aus Base64 speichern (EMPFOHLEN – keine Race Condition)
    if result["screenshots_base64"]["desktop"]:
        save_base64_screenshot(result["screenshots_base64"]["desktop"], "desktop.png")
    if result["screenshots_base64"]["viewport"]:
        save_base64_screenshot(result["screenshots_base64"]["viewport"], "viewport.png")
else:
    # Fehlerfall: HTTP 404
    print(result["message"])
    print(result["images_checked"])

# --- Reiner Screenshot (ohne Bildvergleich) ---
shot = take_screenshot(
    website_url="https://www.maisai.at/fotos/",
    mode="viewport",    # oder "desktop"
    full_page=True,     # gesamte Seite (langes Bild)
)

if shot["success"]:
    # Base64 speichern (EMPFOHLEN)
    save_base64_screenshot(shot["screenshot_base64"], "screenshot.png")

SEARCH-PARAMETER
=================
POST /search  (multipart/form-data)
  url        (string, Pflicht)  – Website-URL zum Durchsuchen
  reference  (file,   Pflicht)  – Referenzbild (JPG/PNG)
  threshold  (int,    default=20) – Max Hash-Distanz (0=identisch, <20=ähnlich)

SCREENSHOT-PARAMETER
=====================
POST /screenshot  (form-data)
  url        (string, Pflicht)    – Website-URL
  mode       (string, default="desktop") – "desktop" oder "viewport"
  scroll_to  (int,    default=0)  – Scroll-Position in Pixeln (0 = Seitenanfang)
  full_page  (bool,   default=false) – Gesamte Seite erfassen (nur bei mode=viewport)

  desktop   = Xvfb-Framebuffer (echte Browseransicht mit Adressleiste)
  viewport  = Playwright-Viewport (nur Seiteninhalt, ohne Browser-Chrome)
  full_page = Ganzseitig: erzeugt ein langes Bild der gesamten Seite (scroll_to wird ignoriert)

CLOUD RUN ENV-VARIABLEN
========================
Auf Cloud Run müssen diese ENV-Variablen gesetzt sein:
  API_KEY          – der API-Key für Authentifizierung
  ALLOWED_ORIGINS  – erlaubte CORS-Origins, kommagetrennt (default: "*")
  IDLE_TIMEOUT     – Sekunden bis Auto-Shutdown (default: 3600)
  SCREEN_WIDTH     – Xvfb-Breite (default: 1920)
  SCREEN_HEIGHT    – Xvfb-Höhe (default: 1080)

Update per gcloud CLI:
  gcloud run services update docker-cloud-app \
    --region=europe-west1 \
    --set-env-vars="API_KEY=<key>,ALLOWED_ORIGINS=https://meineseite.at"

================================================================================
  LOGIN-SESSION MANAGEMENT (für Facebook, Instagram, etc.)
================================================================================

Für Websites die Login-Overlays anzeigen (z.B. Facebook), kann eine persistente
Browser-Session eingerichtet werden. Die Cookies bleiben über Container-Neustarts
erhalten.

WORKFLOW
========
1. Login-Session starten
2. Via noVNC (Port 6080) verbinden und manuell einloggen
3. Login-Session beenden
4. Zukünftige Screenshots nutzen automatisch die gespeicherten Cookies

ENDPOINTS
=========
POST /login-session/start   – Browser öffnen zum manuellen Login
POST /login-session/stop    – Browser schließen (Cookies werden gespeichert)
GET  /login-session/status  – Session-Status und Cookie-Info
DELETE /login-session/clear – Alle Cookies/Profile löschen

BEISPIEL: FACEBOOK-LOGIN
========================

# 1. Login-Session starten
curl -X POST "https://docker-cloud-app-14602204356.europe-west1.run.app/login-session/start" \
  -H "X-API-Key: <API_KEY>" \
  -F "url=https://www.facebook.com/login"

# Antwort:
{
    "success": true,
    "message": "Login-Session gestartet",
    "instructions": [
        "1. Öffne noVNC im Browser: http://<host>:6080",
        "2. Logge dich manuell bei der Website ein",
        "3. Rufe /login-session/stop auf wenn fertig"
    ],
    "url": "https://www.facebook.com/login",
    "novnc_hint": "Verbinde dich via Port 6080 (noVNC)"
}

# 2. Öffne im Browser: http://<host>:6080
#    → Du siehst den Desktop mit Chromium
#    → Logge dich bei Facebook ein
#    → Schließe das Chromium-Fenster NICHT!

# 3. Login-Session beenden
curl -X POST "https://docker-cloud-app-14602204356.europe-west1.run.app/login-session/stop" \
  -H "X-API-Key: <API_KEY>"

# Antwort:
{
    "success": true,
    "message": "Login-Session beendet. Cookies wurden gespeichert."
}

# 4. Session-Status prüfen
curl "https://docker-cloud-app-14602204356.europe-west1.run.app/login-session/status" \
  -H "X-API-Key: <API_KEY>"

# Antwort:
{
    "session_active": false,
    "profile_exists": true,
    "cookies_exist": true,
    "cookies_file_size": 262144,
    "known_logins": ["cookies_present"],
    "profile_path": "/app/data/chrome_profile"
}

# 5. (Optional) Alle Cookies löschen
curl -X DELETE "https://docker-cloud-app-14602204356.europe-west1.run.app/login-session/clear" \
  -H "X-API-Key: <API_KEY>"

DOCKER-COMPOSE VOLUME
=====================
Damit Cookies über Container-Neustarts erhalten bleiben, muss das Volume
gemountet sein:

  volumes:
    - ./data:/app/data

WICHTIG
=======
- Bei Cloud Run: noVNC (Port 6080) ist NICHT erreichbar!
  → Für Cloud Run muss das Profil lokal erstellt und als Volume deployed werden
  → Oder: Separaten Login-Container mit noVNC-Zugang betreiben
- Für lokales Docker: noVNC funktioniert direkt via http://localhost:6080
- Idle-Timeout: Container fährt nach 1h Inaktivität runter (konfigurierbar)
- Cold Start: Erster Request nach Pause dauert ~10-30 Sekunden
- Suchzeit: Typisch 30-60 Sekunden pro Request (timeout=120 empfohlen)
- Kosten: Nur bei Nutzung, $0 wenn idle
- Max Instances: 3 (konfiguriert in deploy.yml)
